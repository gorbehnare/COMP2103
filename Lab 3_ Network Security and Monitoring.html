Part 3: SNORT!

I have installed and enabled an open-source IDS solution called "snort" (see https://www.snort.org/). The server is already configured, so you should not need to modify any settings or configurations.
You are going to simply add your new rules in /etc/snort/rules/local.rules.

After adding your rules you have two options to start testing or applying the rules:

  1. First Option: run the snort console and supply this specific configuration file. This is the better option if you are simply testing, and you will also see errors in your console if you have a syntax error in your file.
    • run: sudo snort -A console -q -i enp0s3 -c /etc/snort/rules/local.rules
    • to exit the console, hit ctrl+C and then Ctrl+Z
  2. Second Option: Apply the configuration by restarting the service
    • sudo service snort restart.
    • The alerts are going to be logged in /var/log/snort/alert. You can simply monitor this file in real-time by having a second ssh session open and using tail -f /var/log/snort/alert.

To test this you have been provided with an example rule:

#  Here is an example: The following rule with log an alert if there are any 
# ICMP packets are detected anywhere. You can use 
# 'tail -f /var/log/snort/alert' and ping the server from your computer to see 
# what the alerts look like. 
alert icmp any any -> any any (msg: "ICMP detected";sid:1000001;rev:1;)
#  Here is an example: The following rule with log an alert if there are any 
# ICMP packets are detected anywhere. You can use 
# 'tail -f /var/log/snort/alert' and ping the server from your computer to see 
# what the alerts look like. 
alert icmp any any -> any any (msg: "ICMP detected";sid:1000001;rev:1;)
  • Notice that since your client is connected to the Internet through the server, any ICMP requests from your client to anywhere will be logged as well, since all traffic is passing through your server (inline monitoring).
  • The sid is just a unique number. Every rule rules must have a unique sid. The best practice is to start from 1,000,000 and above, in case you would like to import rulesets from other sources (in fact the documentation considers any sid less than 1000001 as "reserved").
  • The set of default rules shipped via Ubuntu repositories is still active on this server. Rule sets are available from various sources, some are open and free, and others are provided commercially via subscriptions or for fees.

Snort has been around for a long time, there are many tutorials and demos available that you can find on the internet. The official documentation can be found at https://snort.org/documents#OfficialDocumentation.

  • You may look for the Users manual and Snort rule infographic for a quick start.
  • Notice that we currently have snort version 2 on our system, and not version 3. Version 3 syntax is slightly different and will not work on your lab machine.

Here is a second example snort rule. Try it out by visiting www.facebook.com on the client and checking the logs on the server (you do not need to log in, just visit the domain).

alert tcp any any -> any any (content:"facebook.com"; msg:"Someone Is visiting facebook at this time"; sid:1000009; rev:1;)
